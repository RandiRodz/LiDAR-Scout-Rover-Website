Connection Notes
This page is connection-ready, but it expects a couple of services to be running on the rover host.
rosbridge_server is needed for topic telemetry and the on-page controller.
The rover bringup should already be bridging Point-LIO odometry into /odom, typically from /odom_corrected, before Nav2 and this dashboard come online.
The camera panel prefers /image_raw/compressed through rosbridge, while web_video_server remains an optional fallback on the same host.

How It Connects
No more example input blocks. This is wired around your real rover topics.
/cmd_vel_nav is treated as autonomous driving.
/cmd_vel_avoid is treated as safety override.
/cmd_vel_teleop is the controller topic this page publishes.
/cmd_vel is the final muxed output the rover motor bridge should consume.
